# Security Policy

## Project status

CheckMate is alpha software. It is provided as-is, with no service-level agreements, no guaranteed response times, and no security support commitments.

## Reporting a vulnerability

For non-sensitive issues (input validation, hardening suggestions, dependency hygiene), please open a public GitHub issue at <https://github.com/mandosanti06/CheckMate/issues>.

For sensitive vulnerabilities that should not be disclosed publicly, email the maintainers at <security@example.com>. Include:

- A description of the issue and its impact.
- Steps to reproduce, or a proof-of-concept.
- The affected version (commit hash or release tag).

Please do not file public issues for vulnerabilities that could be actively exploited until a fix is available.

## Supported versions

Only the latest release on `main` receives fixes. Older versions are not patched.
